Ember-Insights.js

The Ember.js framework for measurement of user-centered metrics

View on Github

Introduction

Ember-Insights.js is a Ember based framework that lets you gather user-centered metrics and analyze it through analytic appliances.

The main idea behind is Single Responsibility Principle, and that responsibility for tracking user's metrics should be entirely encapsulated by the different layer from application business logic. The way we do the Ember-Insights.js makes possible to keep business logic and developers entirely encapsulated from tracking user-centered behavior and their analytics logic.

Key features:

  • Ability to track the whole application usage by adding a couple of configuration lines;
  • Ability to specify and track particular actions and transitions;
  • Ability to specify tracking groups and handle them as you wish (all you need to specify handler middleware function);
  • Ability to go with GoogleTracker or plug your own (all you need to extend AbstractTracker interface);
  • Ability to submit user's metrics into multiple analytic appliances on the same time;
  • Designed as an Ember-CLI addon (also available as AMD package);

This project was created at Roundscope Ukraine Labs as a result of our domain experience.
